## 1. 什麼是多型？

多型的意思是「多種形態」，同一個方法名稱在不同情境下有不同的行為。Java 的多型分兩種：

| 類型 | 又稱 | 決定時機 | 實現方式 |
|------|------|---------|---------|
| 編譯時期多型 | 靜態多型 | 編譯時 | 方法多載 Overloading |
| 執行時期多型 | 動態多型 | 執行時 | 方法覆寫 Overriding |

生活類比：「開」這個動作，開門用手推、開車用鑰匙、開電腦按按鈕，同一個動詞在不同對象上有不同的行為。

## 2. 方法多載 Overloading

同一個類別中，方法名稱相同但參數不同。編譯器根據傳入的參數決定呼叫哪一個。

**規則**：
- 方法名稱相同
- 參數數量、型別或順序必須不同
- 回傳值不同不算多載

```java
class Calculator {
    // 兩個整數相加
    int add(int a, int b) {
        return a + b;
    }
    // 三個整數相加（參數數量不同）
    int add(int a, int b, int c) {
        return a + b + c;
    }
    // 兩個小數相加（參數型別不同）
    double add(double a, double b) {
        return a + b;
    }
}
```

**海洋情境範例**：海洋生物的移動方式隨參數不同而變化

```java
class MarineLife {
    String name;
    MarineLife(String name) { this.name = name; }

    // 一般移動
    String move() {
        return name + " 在水中移動";
    }
    // 移動指定距離（多載：參數數量不同）
    String move(int meters) {
        return name + " 移動了 " + meters + " 公尺";
    }
    // 移動到指定深度（多載：參數型別不同）
    String move(double depth) {
        return name + " 下潛到 " + depth + " 公尺深";
    }
}
```

## 3. 方法覆寫 Overriding

子類別重新定義父類別的方法。執行時根據實際物件的類型決定呼叫哪一個。

**規則**：
- 方法名稱、參數、回傳型別必須和父類別相同
- 用 `@Override` 標註，編譯器會幫你檢查
- 不能縮小存取範圍

```java
class MarineLife {
    String name;
    MarineLife(String name) { this.name = name; }

    String move() {
        return name + " 在水中移動";
    }
}

class Fish extends MarineLife {
    Fish(String name) { super(name); }

    @Override
    String move() {
        return name + " 擺動魚鰭快速游動";
    }
}

class Whale extends MarineLife {
    Whale(String name) { super(name); }

    @Override
    String move() {
        return name + " 擺動巨大尾鰭前進";
    }
}

class Jellyfish extends MarineLife {
    Jellyfish(String name) { super(name); }

    @Override
    String move() {
        return name + " 收縮傘狀身體漂浮";
    }
}
```

## 4. 動態方法調派（Runtime Polymorphism 的核心）

父類別的參考變數可以指向子類別的物件，呼叫覆寫方法時，執行的是子類別的版本：

```java
MarineLife[] ocean = {
    new Fish("小丑魚"),
    new Whale("藍鯨"),
    new Jellyfish("水母")
};

for (MarineLife m : ocean) {
    System.out.println(m.move());
}
```

輸出：
```
小丑魚 擺動魚鰭快速游動
藍鯨 擺動巨大尾鰭前進
水母 收縮傘狀身體漂浮
```

這就是多型的威力：同一段程式碼處理不同類型的物件，不需要 if-else 判斷。

## 5. final 關鍵字

`final` 用來「鎖定」，防止被修改：

| 用法 | 效果 | 範例 |
|------|------|------|
| `final` 變數 | 值不能改 | `final double PI = 3.14;` |
| `final` 方法 | 子類別不能覆寫 | `final String type() { return "海洋生物"; }` |
| `final` 類別 | 不能被繼承 | `final class Constants { }` |

```java
class MarineLife {
    final String kingdom() {
        return "動物界";
    }
}

class Fish extends MarineLife {
    // 編譯錯誤！不能覆寫 final 方法
    // @Override String kingdom() { return "..."; }
}
```

## 6. 多載 vs 覆寫 對照表

| 項目 | 多載 Overloading | 覆寫 Overriding |
|------|----------------|----------------|
| 發生位置 | 同一個類別 | 父子類別之間 |
| 方法名稱 | 相同 | 相同 |
| 參數 | 必須不同 | 必須相同 |
| 回傳型別 | 可以不同 | 必須相同 |
| 決定時機 | 編譯時期 | 執行時期 |
| 標註 | 不需要 | 建議加 @Override |
| 用途 | 同功能不同輸入 | 子類別自訂行為 |
